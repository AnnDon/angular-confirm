Angular-confirm

Confirmation modal dialog for AngularJS

Download this project as a .zip file Download this project as a tar.gz file

Usage

This module depends on Angular-UI bootstrap for its $uibModal service and can be used in two different ways: As a service or as a directive.


As a Directive

The confirm directive can be placed on anything that also has ngClick

<button type="button" ng-click="delete()" confirm="Are you sure, {{name}}?">Delete</button>

The confirmIf directive can also be used if you only want the confirmation dialog to show if the expression supplied is true

<button type="button" ng-click="delete()" confirm-if="checked" confirm="Are you sure, {{name}}?">Delete</button>

This button will always show a dialog if the checkbox is checked.
The "OK" button in the dialog will execute the ngClick expression on the button.
Enter your name in the text box to observe it shows up in the confirmation text.

The confirm directive also supports the overwriting of the default labels for the modal title, OK buttion and Cancel button

<button type="button" ng-click="delete()" confirm="Are you sure, {{name}}?" confirm-title="Please verify, {{name}}?" confirm-ok="Yes" confirm-cancel="No">Delete</button>

The confirm directive can also take in modal settings with the confirm-settings attribute

<button type="button" ng-click="delete()" confirm="Are you sure, {{name}}?"  confirm-settings="{size: 'sm'}">Delete</button>

As a Service

$confirm([data], [settings]) returns promise

The $confirm service is available for you to show confirmation dialogs. This allows you to show confirmations without having to have an HTML element tied to the action.

Param Type Details
data Object (Optional) Data to be used by the modal template/templateUrl. Referenced with "data.". The keys "title", "ok", and "cancel" can be used to change the labels used in the modal
settings Object (Optional) Settings that will be passed to $uibModal 
.controller("Controller", function($scope, $confirm) {
  $scope.delete = function() {
      $confirm({text: 'Are you sure you want to delete?'})
        .then(function() {
          $scope.deletedConfirm = 'Deleted';
        });
  }
})

Settings

You can inject and modify $confirmModalDefaults to change the default settings that get passed to the $uibModal service.

.run(function($confirmModalDefaults) {
  $confirmModalDefaults.templateUrl = 'path/to/your/template';
  $confirmModalDefaults.defaultLabels.title = 'Modal Title';
  $confirmModalDefaults.defaultLabels.ok = 'Yes';
  $confirmModalDefaults.defaultLabels.cancel = 'No';
})

You may also pass settings that will override the default settings when using $confirm directly.

$confirm({}, { templateUrl: 'path/to/your/template' })
  .then(function() {
    ...
  });

If you provide your own template or templateUrl you may use any data passed to $confirm by referencing it with data.. The default template expects a "text" key to show the confirmation text.

$confirm({name: "Sally"}, { template: 'Hello {{data.name}}!' })
  .then(function() {
    ...
  });
